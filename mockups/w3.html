FAQ

Common questions

Is Bunle really free?

Yes. All tools (Creator, Viewer, Inspector, CLI, JS SDK) are open source and free forever. Cloud storage starts at 1 GB free. Paid plans add more storage and CDN delivery.

Do recipients need to install anything?

No. Recipients click a link and images stream directly in their browser. They can browse at full resolution or download everything as a ZIP — no account or app needed.

Does Bunle compress my images?

Never. Bunle bundles images byte-for-byte. WebP, JPEG, JXL, PNG — they all pass through untouched. Zero re-encoding, zero quality loss.

Can I self-host Bunle files?

Absolutely. Bunle files are static assets. Host them on S3, R2, Cloudflare, or any CDN. Cloud plans are purely for convenience — the format is open and self-hostable.

How many images can one .bnl file hold?

There's no hard limit on image count. The file size limit depends on your plan: 50 MB per file on Free, 500 MB on Pro, and 2 GB on Business. A typical wedding gallery of 400 photos fits in ~300 MB.

How is Bunle different from a ZIP file?

ZIP requires a full download before you can see anything. Bunle puts the index at byte 0, so images can stream progressively — recipients see photos while the file is still downloading. Plus, Bunle supports random access for jumping to any image instantly.
